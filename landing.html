Repository Insight & Code Structure

Here is a plain English overview of this codebase, detailing the original startup issues and the fixes applied to make the extension operational.

Codebase Structure

  • manifest.json: Defines background service workers and requests permissions for tabs, storage, sidePanel, and scripting.
  • background.ts: Service worker that runs in the background to handle tab state tracking and calls the AI model.
  • App.tsx: React frontend that runs inside Chrome's Side Panel, loading states locally from storage.

Identified Issues & Fixes

  • The Asset Path Issue: Vite build outputs default base path parameters which fail inside extensions. Resolved by pointing inputs directly.
  • Missing helper crash: The front-end referenced a missing `getDomain` method. Fixed by implementing the domain parser locally in App.tsx.
  • No API Key Failures: Built graceful fallback structures so the extension runs locally on Ollama without requiring paid OpenAI keys.
